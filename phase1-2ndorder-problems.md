# Phase 1 — Session 3: 2차 시스템과 damping ratio ζ, natural frequency ωn

> 목표: 1차 시스템(τ, pole 하나)에서 2차 시스템(ζ, ωn, pole 두 개)으로 확장.
> pole이 복소평면 어디 있느냐에 따라 "진동하며 죽는다 / 안 죽는다 / 안 흔들리고 죽는다"가
> 갈리는 걸 손으로 확인하고 눈으로도 본다.
> 선행: phase1-laplace-problems.md (L3의 켤레쌍 pole), phase1-transferfunction-problems.md (T3의 τ↔pole 대응)

## 도구 정리

**표준형(standard form)**:

```
ẍ(t) + 2ζωₙ·ẋ(t) + ωₙ²·x(t) = ωₙ²·u(t)
```

**전달함수**:

```
H(s) = ωₙ² / (s² + 2ζωₙs + ωₙ²)
```

분모 `s² + 2ζωₙs + ωₙ² = 0`을 근의 공식으로 풀면:

```
s = −ζωₙ ± ωₙ√(ζ² − 1)
```

**ζ 값에 따라 근의 성격이 갈린다** (1차 시스템의 pole 하나가, 2차에서는 ζ가 그 모양을 결정하는 다이얼이 됨):

| ζ 범위 | 이름 | pole | 시간응답 모양 |
|---|---|---|---|
| ζ = 0 | undamped | 순허수 ±jωₙ | 영원히 진동 (안 죽음) |
| 0 < ζ < 1 | underdamped | 복소켤레 −ζωₙ ± jωₙ√(1−ζ²) | 진동하며 감쇠 |
| ζ = 1 | critically damped | 중근 −ωₙ | 안 흔들리고 가장 빠르게 감쇠 |
| ζ > 1 | overdamped | 서로 다른 실근 2개 | 안 흔들리고 느리게 감쇠 |

underdamped일 때 실제로 진동하는 주파수는 ωₙ이 아니라:

```
ωd = ωₙ√(1 − ζ²)   (damped frequency, ωₙ보다 항상 작거나 같음)
```

> 표기: ✍️ = 네가 쓴 답 그대로 / 💬 = Claude가 덧붙인 해설·교정

## 문제

### ⚠️ P0. (사용자 제안) H(s) 자체를 유도하기 — L{ẍ} 자력, 대입 과정은 두 군데 교정

문제 제시 전 "표준형 전달함수가 왜 저렇게 나오는지부터 유도해야 하는거 아니야?"라고 먼저 질문 — 좋은 지적, "도구 정리"가 이미 유도된 결과만 던져준 것이었음.

- ✍️ `L{ẍ} = s²X(s) − s·x(0) − ẋ(0)` ← **자력**. L4를 ẋ에 한 번 더 씌우는 방식으로 힌트만 받고 정확히 유도
- ✍️ 표준형 전체에 적용해 H(s)=X(s)/U(s) 구하기 → `X(s) = ωₙ² / ((s+ωₙ)² · s)` → 💬 **❌** 두 가지 오류:
  1. 표준형의 `u(t)`(일반 입력 기호)를 Session 1(L2)의 **unit step u(t)**와 혼동해 U(s)=1/s를 대입함 — H(s)는 입력을 특정하지 않고 U(s)를 심볼로 남겨야 함(T1 방식)
  2. `s² + 2ζωₙs + ωₙ²`를 `(s+ωₙ)²`로 씀 — 이는 ζ=1일 때만 성립하는 특수 케이스
  - 정정: **H(s) = ωₙ² / (s² + 2ζωₙs + ωₙ²)**
- 💬 왜 하필 우변이 ωₙ²인가 (질문 "표준형 오른쪽에 wn은 왜있는거야"): H(0) = DC gain = 계단입력 정상상태값. H(0) 계산 중 처음엔 s=0을 완전히 대입하지 않아 `ωₙ/(2ζs+ωₙ)`로 s가 남은 오답 → 재교정 후 ✍️ **H(0)=1** 자력 도달. 우변이 임의의 b라면 H(0)=b/ωₙ²이 되므로, **H(0)=1(단위 DC gain)이 되려면 b=ωₙ²이어야 함** — 표준형이 ωₙ²을 쓰는 이유(정규화 관례, 물리적 필연 아님)

### ✅ P1. 표준형 맞추기 — 제곱근 빠뜨린 실수 두 차례 자가교정 후 완료

`ẍ + 4ẋ + 3x = 3u(t)` 를 표준형과 비교해서 ωₙ, ζ를 구해라.

- ✍️ 1차 시도: "ζ=4/3, ωₙ=4/√3" → 💬 **❌** 계수 대응 자체가 어긋남
- ✍️ 2차 시도: "ωₙ=3, ζ=4/3" → 💬 **❌** ωₙ²=3에서 **제곱근을 안 씌움** (ωₙ=3이 아니라 √3)
- ✍️ 3차 시도: "ωₙ = ±√3, ζ = 2/ωₙ" ← 대수 정리는 정확. 💬 ωₙ은 물리적으로 항상 양수(주파수)이므로 **ωₙ=√3**으로 확정
- ✍️ 최종수치 계산: **ζ ≈ 1.1547** ← 자력 계산(2/√3), **overdamped** 분류도 자력

### ✅ P2. Pole 구하고 분류하기 — 전부 자력

- ✍️ Pole = **−1, −3** ← 근의 공식 대입, 자력. 원방정식 `s²+4s+3=(s+1)(s+3)` 직접 인수분해와 일치 확인

### ✅ P3. Underdamped 케이스 (ωₙ=2, ζ=0.5) — 자력

- ✍️ Pole = **−1 ± i√3** ← 자력, L3의 켤레쌍 패턴과 동일 구조임을 인지

### ⚠️ P4. Damped frequency — 개념은 해설 필요, 계산·결론은 자력

- 먼저 "wd가 뭐야? 당연히 근의공식 오른쪽 부분이랑 같은 값이겠지"라고 반문 — 대수적으로는 맞지만 **개념(왜 그 값이 "실제 진동 주파수"인지)** 이 빠져있어 💬 해설: pole=−ζωₙ±jωd에서 실수부=envelope 감쇠속도(T4), 허수부=실제 진동수(L3). ωₙ은 ζ=0일 때의 진동수, 감쇠(ζ>0)가 진동을 "끌어당겨 느리게" 만든 결과가 ωd
- ✍️ "같은 ωₙ이어도 실수값(감쇠)이 있으면 실제 주파수는 느려져?" ← 스스로 일반화해 재질문, 정확히 이해했음을 확인
- ✍️ ζ: 0→1일 때 ωd: **ωₙ에서 0으로 감소**, 등호(ωd=ωₙ)는 **ζ=0일 때만** ← 전부 자력

### ⚠️ P5. 감으로 예측하기 — ζ=0/0.5/1은 자력, ζ=2는 두 차례 오답 후 계산으로 자가교정

- ✍️ ζ=0: "코사인처럼 허수축 위아래 대칭 pole 두 개" ← 자력, L3 패턴 정확히 연결
- ✍️ ζ=0.5: "그 pole 두 개가 왼쪽으로 조금 이동" ← 자력
- ✍️ ζ=1: 1차 답 "1이면 1"(불충분) → 재요청 후 "실수축 왼쪽에 있고 진동 없이 감쇠해" ← 자력, 정확
- ✍️ ζ=2: 1차 "발산" → 💬 **❌** (계산 없이 답함, overdamped가 불안정과 무관함을 짚음)
  - 재시도 "더 빠르게 감쇠해" → 💬 아직 계산 안 한 상태의 추측
  - 직접 계산(−4±2√3 = −0.54, −7.46) 후 ✍️ **"더 느리게 감쇠하네"** ← 결론은 정정됐으나 "더 멀리있으니까"라는 근거가 부정확 → 💬 교정: 지배적 pole(−0.54)이 ζ=1의 pole(−2)보다 오히려 **원점에 더 가까움** (더 멀리가 아니라), T3/T4의 "원점에 가까운 pole = 느림" 규칙 재확인
- ✍️ 마무리 질문 "제타 2면 진동하는거 아니야?" ← 좋은 재확인 질문. 💬 해설: 진동 여부는 pole의 **허수부 유무**로 결정, ζ²−1의 부호로 판가름남 (ζ=2 → ζ²−1=3>0 → 실근 → 무진동). 실근/복소근 경계 = 정확히 **ζ=1**
- ✅ Python 검증(phase1_2ndorder_check.py) 실행 후 예측과 일치 확인

## Python 검증 (다 푼 뒤)

```python
import numpy as np
import matplotlib.pyplot as plt
from scipy import signal

wn = 2
zetas = [0, 0.3, 0.7, 1, 2]
fig, ax = plt.subplots(figsize=(8, 5))
for zeta in zetas:
    num = [wn**2]
    den = [1, 2*zeta*wn, wn**2]
    sys = signal.TransferFunction(num, den)
    t, y = signal.step(sys, T=np.linspace(0, 10, 1000))
    ax.plot(t, y, label=f'zeta={zeta}')

ax.axhline(1, color='gray', linestyle=':', linewidth=1)
ax.set_xlabel('t'); ax.set_ylabel('y(t)')
ax.set_title('2nd-order step response by damping ratio')
ax.legend(); ax.grid(True, alpha=0.4)
plt.tight_layout()
plt.show()
```
