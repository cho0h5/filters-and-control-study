# Phase 0 — Session B: 주파수응답 판별법 연습문제

> 목표: 필터 식만 보고 low-pass/high-pass/band-pass를 **원리로** 판별하기
> (재검증에서 틀린 "차분 필터" 문제의 복수전)

## 도구 정리

**핵심 성질**: `x[n] = e^(jωn)`은 지연하면 상수배가 될 뿐 — `x[n−k] = e^(−jωk)·x[n]`

**따라서**: `y[n] = Σ bₖ·x[n−k]` 꼴 필터에 넣으면

```
y[n] = ( Σ bₖ·e^(−jωk) )·x[n]  =  H(e^(jω))·x[n]
        ─────────────────
        frequency response
```

**양 끝 찔러보기**:
- `ω = 0` (DC, 가장 느린 신호): e^0 = 1 → **H(0) = 계수의 합** (= DC gain)
- `ω = π` (가장 빠른 신호 +1,−1,+1,...): e^(−jπk) = (−1)^k → **H(π) = 계수의 교대합**

| H(0) | H(π) | 판정 |
|---|---|---|
| 큼 | ≈0 | low-pass |
| ≈0 | 큼 | high-pass |
| ≈0 | ≈0 | band-pass |

시범 (pure delay): `y[n] = x[n−1]` → H = e^(−jω) → |H| = 1 모든 주파수 → all-pass (위상만 지연)

## 문제

### ⬜ B1. `y[n] = 0.5x[n] + 0.5x[n−1]` (이동평균)

- [ ] H(e^(jω)) =
- [ ] H(0) = 1, 1, 1, 1, ...
- [ ] H(π) = 0, 0, 0, 0, ...
- [ ] 판정: low-pass

> 표기: ✍️ = 내가 쓴 답 그대로 / 💬 = Claude가 덧붙인 해설·교정

### ✅ B1. `y[n] = 0.5x[n] + 0.5x[n−1]` (이동평균) — 💬 전체 함께 풀이 (자력 아님, 복습 대상)

- 💬 H(e^(jω)) = 0.5 + 0.5e^(−jω) — e^(jωn) 꽂고 공통인수로 묶으면 n 없는 덩어리가 남는다
- 💬 H(0) = 1 (계수 합), H(π) = 0 (교대합) → **low-pass**
- 💬 유도 자체를 처음에 못 시작했음 (숫자 표 → 수식 순서로 두 단계에 걸쳐 설명받음)

### ✅ B2. `y[n] = 0.5x[n] − 0.5x[n−1]` (차분 — 설욕 성공 🎉)

- ✍️ H(e^(jω)) = 0.5 − 0.5e^(−jω) ← 자력 유도
- ✍️ H(0) = "0, 0, 0, ..." / H(π) = "1, 1, 1, ..." / 판정: high-pass
- 💬 교정: 판정 자체는 맞음. 단 H는 수열이 아니라 **배율(숫자 하나)** — H(0) = 0, H(π) = 1이 맞는 표기
- 💬 교정: ω=π 입력의 출력은 1,1,1,...이 아니라 **+1,−1,+1,... (진동한 채로 1배 통과)**. 필터는 주파수를 못 바꾸고 크기·위상만 바꾼다

### ✅ B3. `y[n] = (x[n] + x[n−1] + x[n−2]) / 3` (3점 평균) — 자력

- ✍️ H(0) = 1
- ✍️ H(π) = 1/3
- ✍️ 판정: "DC는 그대로 나오고 고주파는 1/3되는 필터"
- 💬 용어 부여: 그걸 **성능 나쁜(leaky) low-pass**라 부름. "차단대역에서 얼마나 덜 새는가" = stopband attenuation, 필터 설계(Phase 3)의 품질 스펙
- ✍️ Python 검증에서 발견: "2π/3에서 0이 되네" — 스스로 찾음
- 💬 정체: 필터의 **zero(영점)**. 세 항 1 + e^(−jω) + e^(−j2ω)가 ω=2π/3에서 정삼각형 벡터 합 = 0. 시간영역: N점 평균은 주기 N 신호를 완전히 죽인다 (60Hz 험 제거에 실전 사용). B3 곡선은 1 → 0(2π/3) → ⅓(π)로 **비단조** — 양끝 찌르기의 한계 재확인

### ✅ B4. `y[n] = 0.25x[n] + 0.5x[n−1] + 0.25x[n−2]` — 자력

- ✍️ H(0) = 1 / H(π) = 0 / B4가 더 좋음
- 💬 맞음: B3은 고주파를 ⅓ 흘리고 B4는 완전히 죽임. 계수 조정으로 성능이 달라지는 것 = "필터 설계"의 씨앗

### ⚠️ B5. (보너스) `y[n] = x[n] − x[n−2]` — 결론 ❌ (계산은 맞음)

- ✍️ H(0) = 0 / H(π) = 0 / "출력이 없음"
- 💬 교정: 잰 건 두 주파수뿐 — 양 끝이 0이어도 중간은 모름. ω=π/2 신호(+1,+1,−1,−1 반복)를 넣으면 x[n−2] = −x[n]이라 y = 2x[n], **2배 증폭!** 양끝 0 + 중간 2 = **band-pass**
- 💬 교훈: 양 끝 찌르기는 1차 판별법일 뿐, 전체 모양은 곡선을 그려야 안다 → Python 검증의 존재 이유

## Python 검증 (다 푼 뒤)

```python
import numpy as np, matplotlib.pyplot as plt
from scipy.signal import freqz

filters = {
    'B1 이동평균': [0.5, 0.5],
    'B2 차분':     [0.5, -0.5],
    'B3 3점평균':  [1/3, 1/3, 1/3],
    'B4':          [0.25, 0.5, 0.25],
    'B5':          [1, 0, -1],
}
for name, b in filters.items():
    w, h = freqz(b, worN=512)
    plt.plot(w, abs(h), label=name)
plt.xlabel('ω (0 → π)'); plt.ylabel('|H|'); plt.legend(); plt.grid(True); plt.show()
```

손으로 구한 H(0), H(π)가 그래프의 양 끝값과 일치하는지 확인할 것.
